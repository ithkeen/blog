开篇第一段直接进入主题：这篇文章为什么值得写，读者可以从中获得什么。

第二段补充背景、问题来源或个人观察。保持具体，避免只写抽象判断。

## 第一个小标题

展开第一个核心观点。可以写问题、判断、例子，或一次实际经历。

如果需要列表，保持简洁：

- 关键点一
- 关键点二
- 关键点三

## 第二个小标题

继续推进文章。每个小标题尽量只承载一个明确主题。

可以加入代码块：

```ts
const idea = "keep it concrete";
```

也可以加入引用：

> 把值得保留的判断写清楚，比收集更多材料更重要。

## 收束

最后一节给出结论、下一步行动，或一个值得继续追问的问题。
